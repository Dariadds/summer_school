# Функциональные требования

> Этап 2. **Что** система должна делать — поведение, функции, бизнес-правила, видимые пользователю сценарии.
> Не путать с нефункциональными: см. [non-functional-requirements.md](non-functional-requirements.md).
>
> Источники: `[CB]` — `0-customer-brief/customer-brief.md`; `[DD]` — `1-elicitation/domain-description.md`; `[CQ:n]` — вопрос/ответ № n в `1-elicitation/customer-questions.md`.

## Шкала приоритетов

| Приоритет | Значение |
| :-- | :-- |
| **P1 — Обязательный (MVP)** | Явно обязателен к запуску ([CQ:33]) |
| **P2 — Высокий** | В скоупе MVP, но не назван среди четырёх обязательных к запуску |
| **P3 — Средний** | Желательный функционал из брифа/уточнений |
| **P4 — Вне MVP** | Явно перенесён или исключён из первой версии |
