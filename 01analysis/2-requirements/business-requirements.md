# Бизнес-требования

> Этап 2. Зачем бизнесу продукт: цели и ограничения верхнего уровня.
> Источники: `[CB]` — `0-customer-brief/customer-brief.md`; `[DD]` — `1-elicitation/domain-description.md`; `[CQ:n]` — вопрос/ответ № n в `1-elicitation/customer-questions.md`.

## Шкала приоритетов

| Приоритет | Значение |
| :-- | :-- |
| **P1 — Обязательный (MVP)** | Явно обязателен к запуску |
| **P2 — Высокий** | В скоупе MVP, но не назван среди четырёх обязательных к запуску |
| **P3 — Средний** | Желательный функционал из брифа/уточнений |
| **P4 — Вне MVP** | Явно перенесён или исключён из первой версии |

## Требования

| ID | Формулировка | Приоритет | Источник |
| :-- | :-- | :-- | :-- |
| **BR-001** | Клиенты должны самостоятельно записываться на заезды через мобильное приложение, без ручной обработки записей владельцем. | P1 | [CB], [DD] |
| **BR-002** | Клиент должен видеть актуальное расписание заездов и свободные места, чтобы выбрать подходящий слот. | P1 | [CB], [DD], [CQ:5], [CQ:33] |
| **BR-003** | Клиент должен управлять своими записями: просматривать бронирования и отменять их по правилам центра. | P1 | [CB], [DD], [CQ:16–18], [CQ:33] |
| **BR-004** | Система должна снижать количество неявок за счёт напоминаний о предстоящей записи. | P1 | [CB], [CQ:25], [CQ:33] |
| **BR-005** | Клиент должен иметь возможность выбрать аренду экипировки (шлем, подшлемник) или использование своей при записи. | P2 | [CB], [DD], [CQ:13–15] |
| **BR-006** | Владелец должен получать обратную связь о работе маршалов через оценки клиентов после заездов. | P3 | [CB], [DD], [CQ:22–24] |
| **BR-007** | При отмене заезда центром (в т.ч. из-за погоды) все затронутые клиенты должны быть своевременно проинформированы. | P1 | [CB], [CQ:27], [CQ:33] |
| **BR-008** | Решение должно быть запущено к началу высокого сезона с ограниченным бюджетом, начиная с минимально необходимого функционала. | P1 | [CB], [CQ:33] |
| **BR-009** | Программа лояльности для постоянных клиентов не входит в первую версию продукта. | P4 | [CB], [CQ:28–30] |

## Ограничения верхнего уровня

- **Срок и бюджет.** Запуск к началу высокого сезона (~2 месяца); бюджет ограничен → MVP-подход. ([CB], [CQ:33])
- **Скоуп поставки.** Только клиентское мобильное приложение и API для него; админка, интерфейсы маршала и серверная логика уже существуют. ([CB], [DD], [CQ:31])
- **Оплата офлайн на старте.** Наличные или перевод; онлайн-оплата — вне первой версии. ([CB], [CQ:19], [CQ:21])
- **Обязательный минимум к запуску.** Просмотр расписания, бронирование, отмена записи, уведомления. ([CQ:33])
