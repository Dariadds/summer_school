# BS-005 · Оценка маршала

| | |
| :-- | :-- |
| **ID** | BS-005 |
| **Тип** | Bottom Sheet |
| **Приоритет** | P3 |
| **Статус** | Черновик |
| **Связанные требования** | FT-026, FT-027, FT-028 · US-015 · UC-007 |

---

## 1. Зачем этот экран

После заезда клиент может оценить работу маршала. Владельцу важно понимать, кто хорошо работает с людьми — клиенту нужен простой способ высказаться без публичного рейтинга.

## 2. Для кого и в какой ситуации

- **Кто:** клиент, завершивший заезд.
- **Когда открывает:** «Оценить маршала» на [SCR-006](SCR-006-booking-details.md); опционально — prompt после заезда.
- **Контекст:** свежие впечатления; может спешить домой.
- **Барьер:** не хочет писать длинный отзыв; боится, что оценка увидят другие.

## 3. Главная задача на экране

**Пользователь должен:** поставить оценку от 1 до 5 звёзд и (опционально) оставить комментарий.

**Вторично может:** пропустить / закрыть без оценки.

## 4. Иерархия информации

| Приоритет | Что показать | Зачем |
| :-- | :-- | :-- |
| 1 | Имя маршала | Кого оцениваем |
| 2 | Шкала 1–5 (звёзды) | Главное действие |
| 3 | Поле комментария (необязательное) | Детали |
| 4 | CTA «Отправить» / «Пропустить» | Завершение |

## 5. Путь пользователя

**Откуда попадает:**
- [SCR-006](SCR-006-booking-details.md) — завершённая бронь
- (Опционально) push/banner после заезда

**Куда уходит после:**
- Отправка → SCR-006 с пометкой «Оценка отправлена»
- Пропуск → SCR-006 без изменений

## 6. Сценарии и состояния экрана

| Ситуация | Что видит пользователь | Что может сделать |
| :-- | :-- | :-- |
| Новая оценка | Звёзды + пустой комментарий | Оценить, отправить |
| Только звёзды | Комментарий пустой | Отправить (валидно) |
| Успех | «Спасибо за отзыв!» | Закрыть |
| Уже оценено | «Вы уже оценили этого маршала» | Закрыть |

## 7. Ключевые элементы интерфейса

- Bottom sheet
- «Как вам маршал [Имя]?»
- Интерактивные 5 звёзд (или аналог)
- Textarea «Комментарий (необязательно)»
- Кнопки «Отправить» и «Пропустить»
- Пояснение «Отзыв видит только администрация центра»

## 8. Тон и тексты (микрокопия)

- **Тон:** лёгкий, благодарный.
- **Примеры:** «Как прошёл заезд с Алексеем?», «Расскажите подробнее (необязательно)», «Спасибо! Это поможет нам стать лучше».
- **Избегать:** «Публичный отзыв», давления «Оцените обязательно».

## 9. Ограничения для макета

- Шкала **строго 1–5**.
- Комментарий **необязателен**.
- Оценки **не показываются другим клиентам** — можно упомянуть в copy.
- Приоритет P3 — можно упростить для MVP.

## 10. Что не входит в этот экран

- Публичный рейтинг маршалов
- Оценка трассы / картов / центра в целом
- Фото к заезду

## 11. Открытые вопросы к дизайнеру

- Звёзды vs emoji vs числовая шкала?
- Максимальная длина комментария?
- Показывать ли шторку автоматически после заезда или только по кнопке?
