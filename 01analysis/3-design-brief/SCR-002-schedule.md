# SCR-002 · Расписание заездов

| | |
| :-- | :-- |
| **ID** | SCR-002 |
| **Тип** | Экран (главная вкладка) |
| **Приоритет** | P1 |
| **Статус** | Черновик |
| **Связанные требования** | FT-001, FT-003, FT-004, FT-005 · US-001 · UC-001 |

---

## 1. Зачем этот экран

Главная точка входа в приложение: клиент видит ближайшие заезды и выбирает подходящий. Заменяет переписку с владельцем в Telegram — всё прозрачно: время, трасса, места, цена, маршал.

## 2. Для кого и в какой ситуации

- **Кто:** потенциальный или постоянный клиент «Апекс».
- **Когда открывает:** ищет когда покататься; планирует выходные; сравнивает слоты.
- **Контекст:** дома вечером или в дороге; хочет быстро «увидеть субботу вечером на длинную трассу».
- **Барьер:** не хочет звонить и ждать ответа; важно сразу видеть свободные места.

## 3. Главная задача на экране

**Пользователь должен:** найти подходящий заезд и открыть его карточку.

**Вторично может:** отфильтровать по датам; перейти в «Мои записи» или «Профиль».

## 4. Иерархия информации

| Приоритет | Что показать | Зачем |
| :-- | :-- | :-- |
| 1 | Дата и время заезда | Когда ехать |
| 2 | Свободные места | Можно ли записаться |
| 3 | Конфигурация трассы + тип (новичковый/опытный) | Подходит ли уровень |
| 4 | Стоимость | Сколько заплатить |
| 5 | Маршал | Кто ведёт заезд |
| 6 | Индикатор активного фильтра дат | Понять, что список отфильтрован |

## 5. Путь пользователя

**Откуда попадает:**
- Старт приложения (авторизованный — вкладка «Расписание»)
- После входа [SCR-001](SCR-001-auth.md)

**Куда уходит после:**

## 6. Сценарии и состояния экрана

| Ситуация | Что видит пользователь | Что может сделать |
| :-- | :-- | :-- |
| Есть заезды | Список карточек на 7 дней, сгруппированных по дням | Открыть заезд, фильтровать |
| Пусто | «Пока нет доступных заездов» | Обновить, расширить фильтр дат |
| Загрузка | Скелетон карточек | Ждать |
| Ошибка сети | Сообщение + «Повторить» | Повторить загрузку |
| Нет мест на заезд | Карточка с «Мест нет», неактивная запись | Только просмотр деталей |
| Заезд отменён центром | Пометка «Отменён центром» | Просмотр без записи |

## 7. Ключевые элементы интерфейса

- Заголовок «Расписание» / «Заезды»
- Кнопка фильтра по датам (с бейджем при активном фильтре)
- Список карточек заездов (скролл)
- Группировка по дням («Сегодня», «Завтра», «Сб, 5 июл»)
- Бейдж типа заезда: новичковый / опытный
- Таб-бар (в авторизованной зоне)
- Pull-to-refresh

## 8. Тон и тексты (микрокопия)

- **Тон:** информативный, энергичный, без перегруза.
- **Примеры:** «Осталось 3 места», «Мест нет», «Короткая трасса · для новичков», «Пока нет доступных заездов — загляните позже».
- **Избегать:** «Слот», «эвент», внутренней терминологии.

## 9. Ограничения для макета

- По умолчанию — 7 ближайших дней.
- Новичковый заезд — до 8 участников, опытный — до 14 (отображается через свободные места от сервера).
- Расписание доступно и до авторизации; запись — только после входа.
- Отменённые центром заезды не скрывать без необходимости — клиент должен понимать, что слот недоступен.

## 10. Что не входит в этот экран

- Фильтр по маршалу, типу трассы (нет в требованиях — только даты)
- Создание/редактирование расписания
- Карта трассы (если нет в требованиях — не добавлять)

## 11. Открытые вопросы к дизайнеру

- Группировка: sticky-заголовки дней или разделители?
- Показывать ли отменённые заезды в общем списке или только в «Мои записи»?
- Нужен ли баннер «Войдите, чтобы записаться» для неавторизованных?

## 12. Связь с требованиями

| Требование | Как отражено на экране |
| :-- | :-- |
| FT-001 | Список на 7 дней по умолчанию |
| FT-003 | Empty state |
| FT-004 | Дата, время, трасса, места, цена, маршал на карточке |
| FT-005 | Тип заезда (новичковый/опытный) |
| US-001 | Быстрый выбор заезда на неделю |
| UC-001 | Просмотр расписания |
