# SCR-001 · Вход / регистрация

| | |
| :-- | :-- |
| **ID** | SCR-001 |
| **Тип** | Экран |
| **Приоритет** | P2 |
| **Статус** | Черновик |
| **Связанные требования** | FT-006, FT-007, FT-008 · US-004, US-005 · UC-003 |

---

## 1. Зачем этот экран

Клиенту нужно подтвердить личность, чтобы оформить запись на заезд. Экран убирает барьер «сложной регистрации»: достаточно телефона и SMS-кода, без паролей и длинных форм.

## 2. Для кого и в какой ситуации

- **Кто:** посетитель картинг-центра «Апекс», уже выбравший заезд или решивший записаться.
- **Когда открывает:** нажал «Записаться» на карточке заезда, будучи не авторизованным.
- **Контекст:** дома или в дороге, хочет быстро завершить запись; может быть первый раз в приложении.
- **Барьер:** не хочет запоминать пароль; боится лишних полей и спама.

## 3. Главная задача на экране

**Пользователь должен:** войти по номеру телефона и подтвердить код из SMS.

**Вторично может:** указать имя при первом входе; вернуться назад к выбранному заезду.

## 4. Иерархия информации

| Приоритет | Что показать | Зачем |
| :-- | :-- | :-- |
| 1 | Поле телефона + кнопка «Получить код» | Главное действие |
| 2 | Поле ввода кода (после отправки SMS) | Подтверждение |
| 3 | Поле имени (только первый вход) | Идентификация в центре |
| 4 | Пояснение «Зачем нужен номер» | Снижает недоверие |

## 5. Путь пользователя

**Откуда попадает:**
- Карточка заезда [SCR-003](SCR-003-ride-details.md) — CTA «Записаться» без авторизации
- (Опционально) любая точка, где нужна авторизация

**Куда уходит после:**
- Успех → продолжение бронирования [SCR-004](SCR-004-booking.md) или возврат к выбранному заезду
- Назад → [SCR-003](SCR-003-ride-details.md)

## 6. Сценарии и состояния экрана

| Ситуация | Что видит пользователь | Что может сделать |
| :-- | :-- | :-- |
| Ввод телефона | Поле +7, кнопка отправки кода | Ввести номер, запросить SMS |
| Ожидание кода | «Код отправлен на …» | Ввести код, запросить повторно |
| Первый вход | Дополнительно поле «Имя» | Ввести имя и завершить вход |
| Неверный код | Сообщение об ошибке | Исправить код |
| Повторная отправка | Таймер до повторной отправки | Подождать или повторить |

## 7. Ключевые элементы интерфейса

- Поле номера телефона с маской
- Кнопка «Получить код» / «Войти»
- Поле одноразового кода (4–6 цифр)
- Поле имени (условно, первый вход)
- Ссылка «Изменить номер»
- Индикатор загрузки при отправке/проверке

## 8. Тон и тексты (микрокопия)

- **Тон:** дружелюбный, короткий, без канцелярита.
- **Примеры:** «Введите номер — пришлём код», «Как к вам обращаться?», «Неверный код, попробуйте ещё раз».
- **Избегать:** «Аутентификация», «Регистрация в системе», пугающих формулировок про персональные данные без пояснения.

## 9. Ограничения для макета

- Только телефон + SMS, без пароля и email.
- Имя и телефон — единственные данные профиля.
- Запись без входа невозможна.
- Клавиатура не должна перекрывать CTA.

## 10. Что не входит в этот экран

- Редактирование профиля после входа → [SCR-007](SCR-007-profile.md)
- Социальные сети / Apple ID / Google
- Согласие на маркетинг

## 11. Открытые вопросы к дизайнеру

- Один экран (телефон → код → имя) или пошаговый wizard?
- Показывать ли логотип «Апекс» и welcome-текст?
- Где разместить поле имени: сразу после кода или отдельным шагом?

## 12. Связь с требованиями

| Требование | Как отражено на экране |
| :-- | :-- |
| FT-006 | Вход по телефону + SMS |
| FT-007 | Поле имени при первом входе |
| FT-008 | Экран обязателен перед бронированием |
| US-004 | Быстрый вход для записи |
| US-005 | Ввод имени |
| UC-003 | Полный сценарий авторизации |
