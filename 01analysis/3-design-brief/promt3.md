# Постановка на дизайн
На основе functional-requirements.md и use-cases.md определи полный список экранов для клиентского приложения, для каждого экрана напиши постановку на дизайн. Каждая постановка должна содержать ID, тип (экран или bottom sheet), приоритет, связанные требования, цель экрана, сценарии и состояния (основной, пусто, ошибка), ключевые элементы, путь пользователя и ограничения из бизнес-правил.
