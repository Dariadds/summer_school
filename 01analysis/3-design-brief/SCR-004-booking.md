# SCR-004 · Оформление бронирования

| | |
| :-- | :-- |
| **ID** | SCR-004 |
| **Тип** | Экран |
| **Приоритет** | P1 |
| **Статус** | Черновик |
| **Связанные требования** | FT-009–FT-012, FT-016 · US-006, US-007 · UC-004 |

---

## 1. Зачем этот экран

Клиент уточняет детали записи: сколько человек едет и нужна ли аренда экипировки. Это последний шаг перед подтверждением — здесь формируется состав группы (себя + друзей/семью до 4 человек).

## 2. Для кого и в какой ситуации

- **Кто:** авторизованный клиент, решивший записаться.
- **Когда открывает:** нажал «Записаться» на [SCR-003](SCR-003-ride-details.md).
- **Контекст:** записывает себя или компанию; может не знать заранее, есть ли своя экипировка у всех.
- **Барьер:** путается в выборе проката; не понимает, можно ли записать друзей без их регистрации.

## 3. Главная задача на экране

**Пользователь должен:** указать количество мест (1–4) и для каждого — своя экипировка или аренда (шлем и/или подшлемник).

**Вторично может:** вернуться к карточке заезда; перейти к подтверждению с итоговой стоимостью.

## 4. Иерархия информации

| Приоритет | Что показать | Зачем |
| :-- | :-- | :-- |
| 1 | Краткое напоминание о заезде (дата, время, трасса) | Контекст |
| 2 | Счётчик мест (1–4) | Размер группы |
| 3 | Выбор экипировки по каждому месту | Прокат vs своё |
| 4 | Предварительная стоимость (если доступна) | Ориентир до подтверждения |
| 5 | CTA «Продолжить» / «Подтвердить» | Следующий шаг |

## 5. Путь пользователя

**Откуда попадает:**
- [SCR-003](SCR-003-ride-details.md) после авторизации

**Куда уходит после:**
- CTA → [BS-002](BS-002-booking-confirm.md) с итоговой стоимостью
- Назад → SCR-003

## 6. Сценарии и состояния экрана

| Ситуация | Что видит пользователь | Что может сделать |
| :-- | :-- | :-- |
| 1 место | Счётчик = 1, один блок экипировки | Выбрать прокат/своё |
| 2–4 места | Несколько блоков «Участник 1…4» | Настроить каждого |
| Нехватка проката | «Прокат недоступен — только со своей экипировкой» | Убрать прокат или отказаться |
| Превышен лимит мест | Счётчик ограничен свободными местами и 4 | Уменьшить число |
| Ошибка | «Не удалось проверить доступность» | Повторить |

## 7. Ключевые элементы интерфейса

- Компактная шапка заезда (дата, время, трасса)
- Stepper / picker количества мест (1–4)
- Для каждого участника:
  - Переключатель «Своя экипировка» / «Нужен прокат»
  - При прокате: чекбоксы «Шлем», «Подшлемник» (можно частично)
- Пояснение: «Друзьям регистрироваться не нужно»
- CTA внизу экрана

## 8. Тон и тексты (микрокопия)

- **Тон:** помогающий, без бюрократии.
- **Примеры:** «Сколько человек?», «Участник 2», «Со своим шлемом», «Взять шлем напрокат», «До 4 человек одной записью».
- **Избегать:** запроса имён/телефонов гостей (не в требованиях).

## 9. Ограничения для макета

- Максимум **4 места** на одну бронь.
- Экипировка: только **шлем** и **подшлемник**; можно выбрать частично.
- Если проката нет — только «своя экипировка».
- После создания брони изменить состав нельзя — не обещать «редактирование».
- Остальные участники не регистрируются в приложении.

## 10. Что не входит в этот экран

- Финальное подтверждение и оплата → [BS-002](BS-002-booking-confirm.md)
- Имена и контакты гостей
- Онлайн-оплата

## 11. Открытые вопросы к дизайнеру

- Один общий выбор экипировки «для всех одинаково» vs настройка по участникам?
- Показывать ли цену проката отдельной строкой до подтверждения?
- Как компактно показать 4 участника на маленьком экране?

## 12. Связь с требованиями

| Требование | Как отражено на экране |
| :-- | :-- |
| FT-009, FT-010 | Выбор 1–4 мест |
| FT-011 | Аренда шлема/подшлемника или своё |
| FT-012 | Блокировка проката при нехватке |
| FT-016 | Нет редактирования после создания |
| US-006, US-007 | Запись компании + экипировка |
| UC-004 | Шаги 2–3 сценария бронирования |
