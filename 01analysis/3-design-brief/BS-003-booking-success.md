# BS-003 · Бронирование оформлено

| | |
| :-- | :-- |
| **ID** | BS-003 |
| **Тип** | Bottom Sheet / модальное окно |
| **Приоритет** | P1 |
| **Статус** | Черновик |
| **Связанные требования** | FT-029 · UC-004 |

---

## 1. Зачем этот экран

Подтвердить клиенту, что запись создана, снять тревогу «а точно записалось?» и направить к следующему шагу — просмотру записи или включению напоминаний.

## 2. Для кого и в какой ситуации

- **Кто:** клиент, только что подтвердивший бронь.
- **Когда открывает:** успешное создание записи после [BS-002](BS-002-booking-confirm.md).
- **Контекст:** облегчение, предвкушение заезда.
- **Барьер:** хочет убедиться в деталях (когда, куда).

## 3. Главная задача на экране

**Пользователь должен:** понять, что запись успешна, и перейти к «Мои записи» или закрыть.

**Вторично может:** разрешить push-уведомления (первый раз).

## 4. Иерархия информации

| Приоритет | Что показать | Зачем |
| :-- | :-- | :-- |
| 1 | Успех: «Вы записаны!» | Эмоциональное подтверждение |
| 2 | Дата, время, трасса | Ключевые детали |
| 3 | CTA «Мои записи» | Следующий шаг |
| 4 | Предложение включить push | Напоминания за 24 ч и 2 ч |

## 5. Путь пользователя

**Откуда попадает:**
- Успешный ответ после BS-002

**Куда уходит после:**
- «Мои записи» → [SCR-005](SCR-005-my-bookings.md) или [SCR-006](SCR-006-booking-details.md)
- Закрытие → [SCR-002](SCR-002-schedule.md)

## 6. Сценарии и состояния экрана

| Ситуация | Что видит пользователь | Что может сделать |
| :-- | :-- | :-- |
| Успех | Иконка успеха + детали | Перейти к записям |
| Push не разрешён | «Включите уведомления, чтобы не забыть» | Перейти в настройки / пропустить |
| Push уже разрешён | Только успех | Закрыть |

## 7. Ключевые элементы интерфейса

- Иконка/иллюстрация успеха
- Заголовок «Вы записаны!»
- Краткая сводка заезда
- Кнопка «Мои записи»
- (Опционально) блок про push-напоминания
- Кнопка «На главную» / закрыть

## 8. Тон и тексты (микрокопия)

- **Тон:** позитивный, короткий.
- **Примеры:** «Отлично! Ждём вас в субботу в 18:00», «Напомним за день и за 2 часа до заезда», «Смотреть мою запись».
- **Избегать:** длинных инструкций; упоминания онлайн-оплаты.

## 9. Ограничения для макета

- Не дублировать полный экран деталей — только ключевое.
- Push — предложить, не блокировать (приложение работает без push).
- Напоминания: за 24 ч и за 2 ч (упомянуть в copy).

## 10. Что не входит в этот экран

- Полные детали брони → SCR-006
- Оплата
- Добавление в календарь (нет в требованиях — опционально как open question)

## 11. Открытые вопросы к дизайнеру

- Bottom sheet vs полноэкранный success?
- Добавить кнопку «Добавить в календарь»?
- Иллюстрация картинга / абстрактная галочка?

| :-- | :-- |
| FT-029 | Предложение push для напоминаний |
