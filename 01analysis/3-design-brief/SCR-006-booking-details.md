# SCR-006 · Детали бронирования

| | |
| :-- | :-- |
| **ID** | SCR-006 |
| **Тип** | Экран |
| **Приоритет** | P1 |
| **Статус** | Черновик |
| **Связанные требования** | FT-021, FT-022, FT-024, FT-030 · US-012, US-014 · UC-005, UC-008 |

---

## 1. Зачем этот экран

Полная информация об одной записи: когда заезд, что забронировано, статус оплаты, можно ли отменить. Сюда же клиент попадает из push при отмене заезда центром.

## 2. Для кого и в какой ситуации

- **Кто:** клиент с конкретной бронью.
- **Когда открывает:** из списка [SCR-005](SCR-005-my-bookings.md); из push «заезд отменён» / «напоминание».
- **Контекст:** готовится к визиту; узнал об отмене; хочет отменить сам.
- **Барьер:** не понимает, вернут ли деньги (MVP — оплата на месте, штрафов нет).

## 3. Главная задача на экране

**Пользователь должен:** понять статус записи и выполнить действие (отменить / оценить маршала / принять информацию об отмене центром).

**Вторично может:** вернуться к списку.

## 4. Иерархия информации

| Приоритет | Что показать | Зачем |
| :-- | :-- | :-- |
| 1 | Статус брони (бейдж) | Текущее состояние |
| 2 | Дата, время, трасса, маршал | Контекст заезда |
| 3 | Количество мест, экипировка | Состав брони |
| 4 | Стоимость + статус оплаты | Финансы |
| 5 | Причина отмены центром (если есть) | Объяснение |
| 6 | CTA «Отменить запись» / «Оценить маршала» | Действия |

## 5. Путь пользователя

**Откуда попадает:**
- [SCR-005](SCR-005-my-bookings.md)
- Push-уведомление (напоминание, отмена центром)
- [BS-003](BS-003-booking-success.md) (опционально)

**Куда уходит после:**
- «Отменить» → [BS-004](BS-004-cancel-confirm.md)
- «Оценить маршала» → [BS-005](BS-005-marshal-rating.md)
- Назад → SCR-005

## 6. Сценарии и состояния экрана

| Ситуация | Что видит пользователь | Что может сделать |
| :-- | :-- | :-- |
| Активная предстоящая | Статус «Активна», кнопка «Отменить» | Отменить |
| < 2 ч до старта | Предупреждение при отмене (через BS-004) | Отменить с предупреждением |
| Отменена клиентом | Статус «Отменена» / «Поздняя отмена» | Только просмотр |
| Отменён центром | Статус + **причина**, нет CTA записи | Прочитать, выбрать другой заезд |
| Завершена | Статус «Завершена», «Оценить маршала» | Оценить |
| Оплата | «Оплачено» / «Не оплачено» | — |

## 7. Ключевые элементы интерфейса

- Хедер «Запись» / «Детали»
- Статус-бейдж (крупный)
- Блок информации о заезде
- Блок состава: N мест, экипировка по участникам
- Строка «Итого: … ₽» + статус оплаты
- Блок причины отмены (условно)
- CTA «Отменить запись» (destructive, только для активных)
- CTA «Оценить маршала» (для завершённых)
- Пояснение «Оплата на месте»

## 8. Тон и тексты (микрокопия)

- **Тон:** информативный; при отмене центром — сочувствующий, без паники.
- **Примеры:** «Заезд отменён из-за дождя. Выберите другую дату», «Оплата при посещении центра», «Бесплатная отмена до … (за 2 часа)».
- **Избегать:** обещаний возврата денег (оплата офлайн).

## 9. Ограничения для макета

- Статус «Отменён центром» с **причиной** обязателен.
- Повторная запись на этот заезд недоступна — не показывать CTA «Записаться снова» на тот же слот.
- Статус оплаты — только отображение (P3).
- Штрафов за позднюю отмену нет — не пугать.

## 10. Что не входит в этот экран

- Сам процесс отмены → BS-004
- Форма оценки → BS-005
- Редактирование брони
- Онлайн-оплата

## 11. Открытые вопросы к дизайнеру

- Показывать ли адрес/место сбора центра (есть в брифе R-015 через API)?
- Countdown «до заезда осталось …»?
- Где разместить «Оценить маршала» — banner или кнопка внизу?

## 12. Связь с требованиями

| Требование | Как отражено на экране |
| :-- | :-- |
| FT-021 | Статус «Отменён центром» + причина |
| FT-022 | Нет повторной записи на отменённый слот |
| FT-024 | Статус оплаты |
| FT-030 | Deep link из push |
| US-012, US-014 | Статус оплаты, отмена центром |
| UC-005, UC-008 | Отмена и обработка отмены центром |
