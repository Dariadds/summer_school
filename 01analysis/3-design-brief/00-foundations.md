# Foundations · Сквозные правила дизайна

> Общие принципы для всех экранов приложения картинг-центра **«Апекс»**.
> Экранные постановки ссылаются сюда и **не дублируют** эти правила.

**Источники:** [CB](../0-customer-brief/customer-brief.md) · [DD](../1-elicitation/domain-description.md) · [ФТ](../2-requirements/functional-requirements.md) · [НФТ](../2-requirements/non-functional-requirements.md)

## 1. Продукт и аудитория

**«Апекс»** — клиентское мобильное приложение для самостоятельной записи на заезды уличного картинг-центра. Заменяет запись через Telegram и доску маркером.

**Единственная роль — «Клиент».** Маршал и владелец работают через существующую инфраструктуру.

**Контекст использования:** клиент записывается с телефона — дома, в дороге, иногда прямо на площадке центра. Нужны крупные элементы, понятные статусы, минимум шагов до записи.

**Оплата:** наличные или перевод на месте; приложение показывает стоимость, но не принимает оплату онлайн (MVP).

## 2. Дизайн-принципы

| # | Принцип | Смысл для макета |
| :-- | :-- | :-- |
| P1 | **Mobile-first** | Одна колонка, крупные зоны нажатия, читаемость на улице и в движении. |
| P2 | **Короткий путь к записи** | От расписания до подтверждения — минимум экранов и полей. |
| P3 | **Понятные правила без давления** | Отмена за 2 часа, поздняя отмена без штрафа — объяснять спокойно, не пугать. |
| P4 | **Честность о доступности** | Свободные места, тип заезда (новичковый/опытный), статус «отменён центром» — видны сразу. |
| P5 | **Только свои данные** | Клиент видит только свои брони; чужие оценки маршалов не показываются. |

## 3. Структурные ориентиры (без бренда)

- Минимальный размер интерактивного элемента — **≥ 44 pt**.
- Основной CTA — на всю ширину, всегда виден (не перекрыт клавиатурой).
- Статусы (активна, отменена, поздняя отмена, отменён центром, оплачено) — **не только цветом**, дублировать текстом/иконкой.
- Ключевые числа (время, свободные места, цена) — визуально доминируют.
- Конкретные цвета, шрифты, иллюстрации — **на усмотрение дизайнера**; бренд «Апекс» в аналитике не зафиксирован.

## 4. Навигация

### 4.1 Зоны приложения

| Зона | Когда | Содержимое |
| :-- | :-- | :-- |
| **Неавторизованная** | До входа | Просмотр расписания возможен; запись требует авторизации |
| **Авторизованная** | После SMS-входа | Полный доступ: запись, мои брони, профиль |

> Уточнение для дизайна: можно показывать расписание до входа, но CTA «Записаться» ведёт на авторизацию ([FT-008](../2-requirements/functional-requirements.md)).

### 4.2 Таб-бар (авторизованная зона)

Три раздела:

| Вкладка | Экран | Назначение |
| :-- | :-- | :-- |
| **Расписание** | [SCR-002](SCR-002-schedule.md) | Главный экран — выбор заезда |
| **Мои записи** | [SCR-005](SCR-005-my-bookings.md) | Список бронирований |
| **Профиль** | [SCR-007](SCR-007-profile.md) | Имя, телефон, выход |

Таб-бар виден на корневых экранах вкладок; на вложенных (карточка заезда, оформление) — скрыт, есть «Назад».

### 4.3 Bottom Sheet

Используется для: фильтр дат, подтверждение брони, успех брони, подтверждение отмены, оценка маршала. Не перегружать полноэкранными формами там, где достаточно шторки.

## 5. Состояния экранов (сквозной паттерн)

| Состояние | Что видит пользователь |
| :-- | :-- |
| **Загрузка** | Скелетон или индикатор; не пустой белый экран |
| **Контент** | Основной сценарий |
| **Пусто** | Понятное сообщение + подсказка (напр. «Пока нет доступных заездов») |
| **Ошибка** | Что случилось + что делать (повторить, вернуться) |

## 6. Статусы бронирования

| Статус | Когда | Акцент в UI |
| :-- | :-- | :-- |
| **Активна** | Запись подтверждена, заезд впереди | Основной, CTA «Отменить» доступен |
| **Отменена** | Клиент отменил заранее (≥ 2 ч) | Нейтральный, в истории |
| **Поздняя отмена** | Клиент отменил < 2 ч до старта | Предупреждающий, без штрафа |
| **Отменён центром** | Заезд отменён (погода и т.д.) | Заметный, с причиной; повторная запись недоступна |
| **Завершена** | Заезд прошёл | В истории; можно предложить оценку маршала |

---

## 7. Тон коммуникации

- **Спокойный, уважительный** — картинг для отдыха, не бюрократия.
- **Прямой** — «Осталось 3 места», «Отмена бесплатна за 2 часа до старта».
- **Без скрытых штрафов** — в MVP штрафов нет; при поздней отмене честно предупредить, что место может не достаться другим.
- **Названия из домена:** заезд, трасса, маршал, экипировка (шлем, подшлемник), бронирование/запись.

---

## 8. Push-уведомления (контекст для UI)

Приложение запрашивает разрешение на push (лучше в момент первой записи, не при старте).

| Событие | Когда | Зачем пользователю |
| :-- | :-- | :-- |
| Напоминание | За 24 ч и за 2 ч до заезда | Не забыть о записи |
| Отмена центром | При отмене заезда | Не приехать зря |

Тап по push → экран деталей бронирования ([SCR-006](SCR-006-booking-details.md)).

## 9. Глоссарий UI

| Термин | Пояснение для интерфейса |
| :-- | :-- |
| **Заезд** | Групповая сессия катания (~15–20 мин + инструктаж) |
| **Конфигурация трассы** | Короткая (новички) или длинная (опытные) |
| **Маршал** | Инструктор на трассе |
| **Место** | Одно место в заезде (до 4 на одну бронь) |
| **Своя экипировка** | Клиент приезжает со шлемом/подшлемником |
| **Прокат** | Аренда шлема и/или подшлемника в центре |

---

## 10. Вне скоупа дизайна MVP

- Онлайн-оплата
- Программа лояльности
- Публичный рейтинг маршалов
- Интерфейсы маршала и администратора
- SMS-напоминания (только push)
