# Шаблон постановки на дизайн экрана

> **Назначение:** единый формат описания экрана для UI/UX-дизайнера.
> Это **не ТЗ на разработку** — не описываем API, поля базы, техническую реализацию, логику бэкенда.
> Фокус: **задача пользователя, контекст, иерархия, поведение, ограничения для макета**.

**Как пользоваться:** скопируйте блок ниже в новый файл `SCR-XXX-name.md` или `BS-XXX-name.md` и заполните все разделы. Сквозные правила — в [00-foundations.md](00-foundations.md); не дублируйте их, только ссылайтесь.

## Чего **не** должно быть в постановке

| Не включать | Почему |
| :-- | :-- |
| Имена полей API, JSON, SQL | Это зона разработки |
| HTTP-коды, алгоритмы, таймауты | Это ТЗ / архитектура |
| Детальные спецификации компонентов кода | Это UI-kit / dev spec |
| Точные hex-цвета и шрифты (если бренд не зафиксирован) | Решает дизайнер |
| Дублирование foundations | Ссылка достаточна |

## Что **должно** быть

- Задача пользователя и контекст использования
- Иерархия информации (что важнее)
- Сценарии и состояния глазами пользователя
- Ограничения из бизнес-правил (лимиты, сроки, запреты)
- Трассировка к FT / US / UC
- Открытые вопросы, где дизайнер принимает решение
