# BS-004 · Подтверждение отмены

| | |
| :-- | :-- |
| **ID** | BS-004 |
| **Тип** | Bottom Sheet |
| **Приоритет** | P1 |
| **Статус** | Черновик |
| **Связанные требования** | FT-018, FT-019, FT-020 · US-010, US-011 · UC-005 |

---

## 1. Зачем этот экран

Клиент отменяет запись — это чувствительное действие. Шторка объясняет последствия: бесплатная отмена за 2 часа до старта vs поздняя отмена без штрафа, но место может не достаться другим.

## 2. Для кого и в какой ситуации

- **Кто:** клиент с активной бронью на [SCR-006](SCR-006-booking-details.md).
- **Когда открывает:** нажал «Отменить запись».
- **Контекст:** планы изменились; может не знать про правило 2 часов.
- **Барьер:** боится штрафа или «чёрного списка» (в MVP штрафов нет).

## 3. Главная задача на экране

**Пользователь должен:** осознанно подтвердить или отказаться от отмены.

**Вторично может:** закрыть шторку и сохранить запись.

## 4. Иерархия информации

| Приоритет | Что показать | Зачем |
| :-- | :-- | :-- |
| 1 | Тип отмены: ранняя или поздняя | Главное последствие |
| 2 | Пояснение правила 2 часов | Прозрачность |
| 3 | CTA «Да, отменить» / «Оставить запись» | Решение |

## 5. Путь пользователя

**Откуда попадает:**
- [SCR-006](SCR-006-booking-details.md) — «Отменить запись»

**Куда уходит после:**
- Подтверждение → SCR-006 или SCR-005 со статусом «Отменена» / «Поздняя отмена»
- Отмена действия → SCR-006 без изменений

## 6. Сценарии и состояния экрана

| Ситуация | Что видит пользователь | Что может сделать |
| :-- | :-- | :-- |
| Ранняя отмена (≥ 2 ч) | «Место освободится для других» | Подтвердить отмену |
| Поздняя отмена (< 2 ч) | **Предупреждение:** место может не достаться другим, штрафа нет | Подтвердить / передумать |
| Загрузка | Индикатор | Ждать |
| Ошибка | «Не удалось отменить» | Повторить |

## 7. Ключевые элементы интерфейса

- Bottom sheet
- Заголовок «Отменить запись?»
- Текст-пояснение (вариант для ранней / поздней отмены)
- Кнопка destructive «Отменить запись»
- Кнопка secondary «Оставить запись»
- (Опционально) время до старта: «До заезда 1 ч 20 мин»

## 8. Тон и тексты (микрокопия)

- **Тон:** спокойный, честный, без давления.
- **Ранняя:** «Место станет доступно для других клиентов».
- **Поздняя:** «До заезда меньше 2 часов. Штрафа не будет, но место может простоять. Всё равно отменить?»
- **Избегать:** «Штраф», «П penalty», запугивания.

## 9. Ограничения для макета

- Граница: **ровно 2 часа** — ещё ранняя отмена.
- Поздняя отмена: предупреждение **обязательно** (FT-020).
- Штрафов в MVP **нет** — явно не упоминать деньги.
- Отмена только **целиком** (вся бронь).

## 10. Что не входит в этот экран

- Частичная отмена одного места
- Возврат денег (оплата на месте)
- Причина отмены от клиента (не требуется)

## 11. Открытые вопросы к дизайнеру

- Два разных текста в одной шторке или автоматический выбор по времени?
- Нужен ли checkbox «Я понимаю последствия» при поздней отмене?
- Красная кнопка vs outline для destructive action?

## 12. Связь с требованиями

| Требование | Как отражено на экране |
| :-- | :-- |
| FT-018 | Ранняя отмена ≥ 2 ч, место освобождается |
| FT-019 | Поздняя отмена без штрафа |
| FT-020 | Предупреждение перед поздней отменой |
| US-010, US-011 | Отмена и предупреждение |
| UC-005 | Сценарий отмены |
