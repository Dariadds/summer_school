# BS-001 · Фильтр по датам

| | |
| :-- | :-- |
| **ID** | BS-001 |
| **Тип** | Bottom Sheet |
| **Приоритет** | P2 |
| **Статус** | Черновик |
| **Связанные требования** | FT-002 · US-002 · UC-001 |

---

## 1. Зачем этот экран

По умолчанию расписание показывает 7 дней. Клиенту нужно иногда смотреть дальше — например, спланировать заезд через две недели. Фильтр расширяет горизонт без перегрузки главного экрана.

## 2. Для кого и в какой ситуации

- **Кто:** клиент, просматривающий расписание.
- **Когда открывает:** нажал «Фильтр» / иконку календаря на [SCR-002](SCR-002-schedule.md).
- **Контекст:** планирует заранее; не нашёл подходящий заезд в ближайшую неделю.
- **Барьер:** не хочет листать бесконечный список — нужен понятный диапазон.

## 3. Главная задача на экране

**Пользователь должен:** выбрать период дат и применить фильтр к списку заездов.

**Вторично может:** сбросить фильтр к дефолтным 7 дням.

## 4. Иерархия информации

| Приоритет | Что показать | Зачем |
| :-- | :-- | :-- |
| 1 | Выбор «с / по» или календарный диапазон | Главный контрол |
| 2 | Кнопка «Применить» | Подтверждение |
| 3 | «Сбросить» | Вернуться к 7 дням |
| 4 | Подсказка текущего диапазона | Обратная связь |

## 5. Путь пользователя

**Откуда попадает:**
- [SCR-002](SCR-002-schedule.md) — кнопка фильтра

**Куда уходит после:**
- «Применить» / закрытие → обновлённый [SCR-002](SCR-002-schedule.md)
- «Сбросить» → дефолт 7 дней на SCR-002

## 6. Сценарии и состояния экрана

| Ситуация | Что видит пользователь | Что может сделать |
| :-- | :-- | :-- |
| Первое открытие | Дефолтный диапазон (7 дней) или пустой выбор | Задать даты |
| Выбран диапазон | Даты «с» и «по» | Применить |
| Некорректный диапазон | «Дата окончания раньше начала» | Исправить |
| Сброс | Дефолт 7 дней | Применить |

## 7. Ключевые элементы интерфейса

- Bottom sheet с handle
- Два поля даты: «С» и «По» (или inline-календарь)
- Кнопки «Применить» и «Сбросить»
- Крестик закрытия

## 8. Тон и тексты (микрокопия)

- **Тон:** нейтральный, подсказывающий.
- **Примеры:** «Период», «Показать заезды с … по …», «Сбросить фильтр».
- **Избегать:** сложных пресетов, которых нет в требованиях («Выходные», «Эта неделя» — опционально, не обязательно).

## 9. Ограничения для макета

- **Только фильтр по датам** — без фильтра по маршалу, типу трассы (нет в требованиях).
- Границы дат включительные.
- После применения на SCR-002 виден индикатор активного фильтра.

## 10. Что не входит в этот экран

- Фильтр «только со свободными местами»
- Фильтр по маршалу или конфигурации трассы
- Сортировка

## 11. Открытые вопросы к дизайнеру

- Календарь в шторке или нативный date picker?
- Быстрые пресеты («+7 дней», «+14 дней») — нужны ли?
- Максимальный диапазон (ограничить месяцем)?

## 12. Связь с требованиями

| Требование | Как отражено на экране |
| :-- | :-- |
| FT-002 | Фильтр диапазона дат за пределами 7 дней |
| US-002 | Поиск заезда дальше недели |
| UC-001 | Расширение горизонта расписания |
