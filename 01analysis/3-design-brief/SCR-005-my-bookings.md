# SCR-005 · Мои бронирования

| | |
| :-- | :-- |
| **ID** | SCR-005 |
| **Тип** | Экран (вкладка таб-бара) |
| **Приоритет** | P1 |
| **Статус** | Черновик |
| **Связанные требования** | FT-017 · US-009 · UC-006 |

---

## 1. Зачем этот экран

Клиенту нужно видеть все свои записи — предстоящие и прошедшие — в одном месте. Это точка контроля: когда заезд, какой статус, можно ли отменить.

## 2. Для кого и в какой ситуации

- **Кто:** авторизованный клиент с хотя бы одной записью (или зашедший проверить).
- **Когда открывает:** вкладка «Мои записи»; после успешного бронирования; получил push.
- **Контекст:** уточняет планы; готовится к заезду; проверяет прошлые визиты.
- **Барьер:** боится, что запись «потерялась».

## 3. Главная задача на экране

**Пользователь должен:** найти нужную бронь и открыть её детали.

**Вторично может:** перейти к расписанию для новой записи.

## 4. Иерархия информации

| Приоритет | Что показать | Зачем |
| :-- | :-- | :-- |
| 1 | Предстоящие записи (ближайшие сверху) | Актуальное |
| 2 | Статус брони | Активна / отменена / отменён центром |
| 3 | Дата, время, трасса | Идентификация записи |
| 4 | Прошедшие записи (ниже или отдельная секция) | История |
| 5 | CTA «Записаться на заезд» при пустом списке | Новая запись |

## 5. Путь пользователя

**Откуда попадает:**
- Таб-бар «Мои записи»
- [BS-003](BS-003-booking-success.md)
- Push-уведомление (→ обычно сразу SCR-006)

**Куда уходит после:**
- Тап по записи → [SCR-006](SCR-006-booking-details.md)
- «Записаться» → [SCR-002](SCR-002-schedule.md)

## 6. Сценарии и состояния экрана

| Ситуация | Что видит пользователь | Что может сделать |
| :-- | :-- | :-- |
| Есть предстоящие | Карточки с датой, статусом | Открыть детали |
| Только прошлые | Секция «Прошедшие» | Открыть, оценить маршала |
| Пусто | «У вас пока нет записей» + CTA | Перейти к расписанию |
| Отменён центром | Заметный статус + причина (кратко) | Открыть детали |
| Загрузка / ошибка | Скелетон / повторить | Обновить |

## 7. Ключевые элементы интерфейса

- Заголовок «Мои записи»
- Секции «Предстоящие» / «Прошедшие» (или табы)
- Карточка брони: дата, время, трасса, статус-бейдж, число мест
- Таб-бар
- Empty state с CTA

## 8. Тон и тексты (микрокопия)

- **Тон:** спокойный, статусный.
- **Примеры:** «Предстоящие», «Завершённые», «Отменён центром · дождь», «2 места».
- **Статусы:** «Активна», «Отменена», «Поздняя отмена», «Отменён центром», «Завершена».

## 9. Ограничения для макета

- Статусы по [foundations §6](00-foundations.md) — не только цветом.
- Отменённые центром записи **не исчезают** из списка.
- Клиент видит **только свои** брони.

## 10. Что не входит в этот экран

- Отмена (действие) → [SCR-006](SCR-006-booking-details.md) + [BS-004](BS-004-cancel-confirm.md)
- Оценка маршала → [BS-005](BS-005-marshal-rating.md)
- Редактирование состава брони

## 11. Открытые вопросы к дизайнеру

- Табы «Предстоящие / Прошедшие» vs одна лента с разделителем?
- Показывать ли статус оплаты на карточке списка?
- Badge «Оцените маршала» на прошедших без оценки?
