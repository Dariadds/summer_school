# Test Cases — Feature 1: Оценка инструктора

## Замечания
- В текущей реализации кнопка на экране деталей записи отображается как "Оценить маршала", хотя в спецификации речь идёт об оценке инструктора.
- Оценка сохраняется локально и не должна отправляться повторно для той же записи.

### TC-RATING-001

**Feature:** feature-1-rating

**Название**
Успешная оценка инструктора после завершённого бронирования

**Цель**
Проверить, что пользователь может поставить оценку инструктору, сохранить её и увидеть после повторного открытия деталей бронирования.

**Предусловия**
- Пользователь авторизован в приложении.
- В профиле есть завершённая запись, по которой можно оставить оценку.
- На устройстве доступно локальное хранилище приложения.

**Шаги**
1. Открыть экран деталей бронирования для завершённой записи.
2. Нажать кнопку "Оценить инструктора".
3. Выбрать 5 звёзд.
4. Ввести комментарий "Спасибо за занятие".
5. Нажать "Отправить".
6. Закрыть и снова открыть экран деталей этой же записи.

**Ожидаемый результат**
- Открывается модальное окно для оценки.
- После отправки оценка сохраняется локально.
- При повторном открытии записи отображается сохранённая оценка, а повторная отправка не требуется.

**Приоритет**
High

**Тип**
Functional

### TC-RATING-002

**Feature:** feature-1-rating

**Название**
Повторная отправка оценки для одной записи не допускается

**Цель**
Проверить, что для уже оценённой записи повторная оценка не создаётся и не отображается повторно.

**Предусловия**
- Пользователь открыл запись, по которой уже сохранена оценка.
- Оценка уже присутствует в локальном хранилище приложения.

**Шаги**
1. Открыть экран деталей уже оценённой записи.
2. Проверить наличие кнопки оценки.
3. Попытаться открыть механизм оценки, если интерфейс позволяет это сделать.

**Ожидаемый результат**
- Повторная оценка не создаётся.
- Интерфейс не допускает дублирования оценки для той же записи.
- Пользователь видит состояние, соответствующее уже сохранённой оценке.

**Приоритет**
Medium

**Тип**
Negative

### TC-RATING-003

**Feature:** feature-1-rating

**Название**
Граничный сценарий: минимальная оценка без комментария

**Цель**
Проверить, что минимальная оценка и пустой комментарий корректно обрабатываются.

**Предусловия**
- Пользователь авторизован.
- Есть завершённая запись без сохранённой оценки.

**Шаги**
1. Открыть экран деталей бронирования.
2. Нажать кнопку "Оценить инструктора".
3. Выбрать 1 звезду.
4. Оставить поле комментария пустым.
5. Нажать "Отправить".

**Ожидаемый результат**
- Оценка отправляется успешно.
- Пустой комментарий принимается как допустимое значение.
- Состояние записи обновляется так, что повторная оценка больше не доступна.

**Приоритет**
Medium

**Тип**
Regression
