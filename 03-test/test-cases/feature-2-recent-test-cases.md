# Test Cases — Feature 2: Недавние маршруты

## Замечания
- В текущей реализации секция "Недавние" выводится на экране списка маршрутов, а история сохраняется локально.
- В UI сейчас показывается до 3 последних маршрутов, тогда как в документации указано хранение до 5; в тест-кейсе ниже проверяется требование по документации.

### TC-RECENT-001

**Feature:** feature-2-recent

**Название**
Успешное сохранение и открытие недавнего маршрута

**Цель**
Проверить, что после просмотра деталей маршрута он появляется в разделе "Недавние" и открывается по тапу.

**Предусловия**
- Пользователь авторизован.
- На устройстве доступна локальная история маршрутов.
- В списке доступен хотя бы один маршрут.

**Шаги**
1. Открыть экран списка маршрутов.
2. Выбрать маршрут и открыть его детали.
3. Дождаться загрузки деталей.
4. Вернуться на экран списка маршрутов.
5. Найти секцию "Недавние".
6. Нажать на сохранённый маршрут.

**Ожидаемый результат**
- Маршрут появляется в секции "Недавние".
- При нажатии открываются детали выбранного маршрута.
- История сохранена локально и доступна после возврата на экран списка.

**Приоритет**
High

**Тип**
Functional

### TC-RECENT-002

**Feature:** feature-2-recent

**Название**
Повторный просмотр одного маршрута не создаёт дубликаты

**Цель**
Проверить, что при повторном просмотре маршрута в истории не появляется дубликат, а элемент перемещается в начало списка.

**Предусловия**
- В истории уже есть хотя бы один маршрут.
- Пользователь может открыть маршрут повторно.

**Шаги**
1. Открыть экран списка маршрутов.
2. Открыть один и тот же маршрут дважды подряд.
3. Вернуться на экран списка маршрутов.
4. Проверить список раздела "Недавние".

**Ожидаемый результат**
- В истории не появляется дубликат этого маршрута.
- Маршрут отображается один раз.
- При повторном просмотре он становится самым свежим элементом в списке.

**Приоритет**
Medium

**Тип**
Regression

### TC-RECENT-003

**Feature:** feature-2-recent

**Название**
Граничный сценарий: история не должна содержать более 5 маршрутов

**Цель**
Проверить, что в истории сохраняется не более 5 последних уникальных маршрутов.

**Предусловия**
- Пользователь может последовательно открывать несколько разных маршрутов.
- История маршрутов очищена или находится в стартовом состоянии.

**Шаги**
1. Открыть 6 разных маршрутов по очереди.
2. После каждого открытия возвращаться на экран списка маршрутов.
3. Проверить содержимое раздела "Недавние".

**Ожидаемый результат**
- В разделе "Недавние" отображается не более 5 последних уникальных маршрутов.
- Самые старые маршруты из истории не отображаются.

**Приоритет**
Medium

**Тип**
Boundary
