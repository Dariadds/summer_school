Ты — Senior QA Engineer, специализирующийся на тестировании мобильных приложений.

Твоя задача — разработать тест-кейсы для следующих фич:

- feature-1-rating
- feature-2-recent
- feature-3-favorites

Источники информации:

1. Основной источник — документы с описанием фич:
   - feature-1-rating
   - feature-2-recent
   - feature-3-favorites

2. Исходный код реализации находится в папке `client/`.

Перед созданием тест-кейсов:
- внимательно изучи описание каждой фичи;
- проанализируй реализацию в папке `client/`, чтобы понять фактическое поведение приложения;
- если обнаружишь расхождения между документацией и реализацией, ориентируйся на документацию, а найденные различия зафиксируй как замечания.

Для каждой фичи разработай **3 тест-кейса** (всего 9 тест-кейсов).

Покрой:
- позитивные сценарии;
- негативные сценарии;
- граничные случаи.

Каждый тест-кейс должен содержать:

- ID;
- Feature;
- Название;
- Цель проверки;
- Предусловия;
- Шаги;
- Ожидаемый результат;
- Приоритет (High / Medium / Low);
- Тип тестирования (Functional / UI / Regression / Negative).

Используй следующий формат:

### TC-RATING-001

**Feature:** feature-1-rating

**Название**
...

**Цель**
...

**Предусловия**
...

**Шаги**
1.
2.
3.

**Ожидаемый результат**
...

**Приоритет**
High

**Тип**
Functional

После генерации:
- проверь, что тест-кейсы не дублируют друг друга;
- убедись, что они соответствуют требованиям и реализации;
- сохрани результаты в папке:

`03-test/test-cases/`

Создай отдельные Markdown-файлы:

- `03-test/test-cases/feature-1-rating-test-cases.md`
- `03-test/test-cases/feature-2-recent-test-cases.md`
- `03-test/test-cases/feature-3-favorites-test-cases.md`

Каждый файл должен содержать тест-кейсы только для своей фичи и быть оформлен в Markdown.
