# Use cases / сценарии

> Этап 2. Сценарии использования. Обязательно прорабатываем **альтернативные потоки** —
> ИИ любит их забывать.
>
> **Скоуп: клиентское приложение и API для него.** Слоты, маршруты и инструкторы поступают
> из существующего бэкенда через API (read-only).

## UC-1: Запись на прогулку (себя и гостей)

- **Актёры:** Клиент
- **Предусловия:** Клиент авторизован; в расписании есть хотя бы один слот со свободными местами.
- **Триггер:** Клиент хочет забронировать место на прогулке.

### Основной поток
1. Клиент открывает список слотов.
2. (Опц.) Применяет фильтры: дата/период, тип маршрута, наличие мест, инструктор.
3. Открывает карточку слота и просматривает параметры (дата/время, маршрут, инструктор, цена, свободно мест, доступность прокатных досок).
4. Указывает число мест (себя + до 2 гостей).
5. Для каждого места выбирает вариант доски: своя / прокатная.
6. Подтверждает запись.
7. Система проверяет лимиты (`min(свободные доски, потолок маршрута)` и свободный прокатный фонд), создаёт запись, уменьшает счётчики свободных мест/прокатных досок.
8. Система показывает подтверждение с ценой (оплата офлайн) и добавляет запись в «Мои бронирования».

### Альтернативные потоки
- **A1. Все доски — свои.** На шаге 5 прокатный фонд не затрагивается; уменьшается только число мест в группе.
- **A2. Часть мест — прокат.** На шаге 7 проверяется и уменьшается прокатный фонд только на число «прокатных» мест.
- **A3. Фильтр не дал результатов.** На шаге 2 список пуст — система предлагает сбросить/изменить фильтры.

### Исключения / ошибки
- **E1. Не хватает мест в группе.** Запрошено больше, чем свободно мест → запись отклоняется, показывается актуальное число свободных мест.
- **E2. Не хватает прокатных досок.** Прокатных мест запрошено больше, чем свободно досок → система предлагает уменьшить число прокатных мест или выбрать «своя доска».
- **E3. Гонка запросов / слот заполнился параллельно.** На шаге 7 места уже заняты другим клиентом → запись отклоняется без овербукинга и двойной брони (NFR-8), список обновляется.
- **E4. Сетевой сбой при подтверждении.** Запись не создаётся частично; повтор операции не приводит к задвоению (NFR-9).

### Постусловия
- При успехе: создана запись клиента; счётчики слота и прокатного фонда уменьшены; запись видна в «Мои бронирования».
- При ошибке: состояние слота не изменилось.

## UC-2: Отмена записи

- **Актёры:** Клиент
- **Предусловия:** У клиента есть предстоящая запись; прогулка ещё не стартовала.
- **Триггер:** Клиент решает отменить участие.

### Основной поток
1. Клиент открывает «Мои бронирования» и выбирает предстоящую запись.
2. Нажимает «Отменить».
3. Система определяет время до старта.
4. **Ранняя отмена (≥2 ч до старта):** запись помечается «отменена», забронированные места и прокатные доски возвращаются в слот.
5. Система показывает подтверждение отмены.

### Альтернативные потоки
- **A1. Поздняя отмена (<2 ч до старта).** На шаге 4 запись помечается «поздняя отмена»; места НЕ освобождаются; денежных штрафов нет; клиенту показывается пояснение политики 2 часов.

### Исключения / ошибки
- **E1. Прогулка уже стартовала/прошла.** Кнопка отмены недоступна; запись нельзя отменить.
- **E2. Запись уже отменена.** Повторная отмена не выполняется; показывается текущий статус.

### Постусловия
- Статус записи обновлён («отменена» / «поздняя отмена»).
- При ранней отмене места и прокатные доски снова доступны другим клиентам.

## UC-3: Просмотр и фильтрация списка слотов

- **Актёры:** Клиент
- **Предусловия:** Клиент авторизован.
- **Триггер:** Клиент хочет найти подходящую прогулку.

### Основной поток
1. Клиент открывает список слотов (по умолчанию — предстоящие со свободными местами).
2. Задаёт фильтры: дата/период, тип маршрута, наличие мест, инструктор.
3. Система показывает отфильтрованный список с ключевыми параметрами каждого слота.

### Альтернативные потоки
- **A1. Сброс фильтров.** Клиент очищает фильтры → возвращается полный список.
- **A2. Несколько фильтров одновременно.** Условия комбинируются по «И».

### Исключения / ошибки
- **E1. Нет слотов под условия.** Показывается пустое состояние с подсказкой изменить фильтры.

### Постусловия
- Клиент видит актуальный список доступных слотов; данные расписания не изменяются.
