# Функциональные требования

> Этап 2. Что система должна делать (функции, поведение).
> Не путать с нефункциональными ([non-functional-requirements.md](non-functional-requirements.md)).

Приоритеты по MoSCoW: **Must** — обязательно к запуску; **Should** — желательно при наличии времени; **Could** — приятное дополнение; **Won't** — вне границ скоупа.

## Границы скоупа (клиентское приложение + API)

Скоуп — **клиентское мобильное приложение и API для него**. Функции инструктора, владельца и
администратора обеспечиваются **существующей инфраструктурой** и здесь не описываются.

Следствия для скоупа:

- **Слоты (эвенты) приходят из существующего бэкенда через API.** Экранов создания и редактирования расписания в клиентском приложении нет.
- **Маршруты и инструкторы** доступны клиенту как справочные (read-only) данные из API — клиент видит их в карточке слота, но не управляет ими.

## Запись на эвент (клиент)

| ID | Требование | Приоритет | Источник |
| :-- | :-- | :-- | :-- |
| FR-10 | Система должна позволять клиенту записаться на выбранный слот. | Must | Бриф; описание домена |
| FR-11 | Система должна позволять клиенту при записи выбрать вариант доски: своя или прокатная. | Must | Бриф; ответ №2 |
| FR-12 | Система должна позволять клиенту одной записью забронировать несколько мест (себя и +1–2 гостей) при наличии свободных мест. | Must | Ответ №3 |
| FR-13 | Система должна ограничивать число доступных мест слота величиной `min(свободные доски, потолок маршрута)`, где потолок новичкового маршрута ≤8, опытного ≤12. | Must | Ответ №1; описание домена (ограничения) |
| FR-14 | Система должна вести отдельный учёт прокатного фонда (12 досок): запись «своя доска» занимает место в группе, но не занимает прокатную доску. | Must | Ответ №2; глоссарий |
| FR-15 | Система должна запрещать запись, превышающую лимит мест или лимит свободных прокатных досок, исключая двойную бронь и овербукинг. | Must | Бриф (двойные записи); ответ №2 |

> **Вне скоупа.** Функции инструктора, владельца/администратора, расписание-CRUD, оценки

> инфраструктурой или относятся к развитию — в клиентском приложении и его API не реализуются.
